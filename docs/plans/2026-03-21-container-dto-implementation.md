# Container DTO Implementation Plan (v2 — post-audit)

> **For Agent:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan task-by-task.

**Goal:** Replace `NormalizedResult<TRoot>` with a generated container DTO (`Normalized{RootType}Result`) that holds `RootIndex` and `{TypeName}List` arrays for ALL types in the normalization graph.

**Architecture:** New `ContainerEmitter` generates the container class. Per-type DTOs are generated for ALL types (including root) — unchanged from today. `NormalizerEmitter` returns the container. `DenormalizerEmitter` accepts the container. `NormalizedResult<TRoot>` is deleted. `NormalizationContext` stays internal (used during normalization, then discarded).

**Key design change from v1:** The container is a **separate class** from per-type DTOs. The root type gets a per-type DTO AND appears in the container's entity lists. `RootIndex` points into the root's list. This avoids recursive type references when root IS the entity type (which is 100% of existing usage).

**Tech Stack:** C# 12, Roslyn incremental source generators, NUnit 4, CSharpier formatting.

**Design doc:** `docs/plans/2026-03-21-container-dto-design.md` (note: design doc reflects v1 "root dissolves" approach — this plan supersedes it)

**Skills:** @source-generator-dev @dotnet-tdd @csharpier @dotnet-architecture

## Revised Container Shape

For `NormalizeGraph<Person>()` where Person has `Address Home`:

```csharp
// Per-type DTOs (unchanged, ALL types including root)
public partial class NormalizedPerson : IEquatable<NormalizedPerson>
{
    public string Name { get; set; } = default!;
    public int HomeIndex { get; set; }
}
public partial class NormalizedAddress : IEquatable<NormalizedAddress>
{
    public string City { get; set; } = default!;
}

// Container (separate class, generated by ContainerEmitter)
public partial class NormalizedPersonResult
{
    public int RootIndex { get; set; }
    public NormalizedPerson[] PersonList { get; set; } = Array.Empty<NormalizedPerson>();
    public NormalizedAddress[] AddressList { get; set; } = Array.Empty<NormalizedAddress>();
}
```

Serialized JSON:
```json
{
  "RootIndex": 0,
  "PersonList": [
    { "Name": "Alice", "HomeIndex": 0 },
    { "Name": "Bob", "HomeIndex": 0 }
  ],
  "AddressList": [
    { "City": "Seattle" }
  ]
}
```

### Task 2: Modify NormalizerEmitter to return container

**Files:**
- Modify: `src/DataNormalizer.Generators/Emitters/NormalizerEmitter.cs` (EmitPublicNormalizeMethod only — lines 57-79)
- Modify: `tests/DataNormalizer.Generators.Tests/Emitters/NormalizerEmitterTests.cs`

The public `Normalize()` method changes return type from `NormalizedResult<{DtoFullName}>` to the container type (`Normalized{TypeName}Result`). The internal structure stays the same — root type still gets a `NormalizeXxx()` helper (NO skipping, critical for self-referencing roots like TreeNode).

**Key changes to EmitPublicNormalizeMethod (lines 57-79):**

Before:
```csharp
public static NormalizedResult<{dtoFullName}> Normalize({rootType} source)
{
    var context = new NormalizationContext({typeCount});
    var rootIndex = Normalize{TypeName}(source, context);
    var root = context.GetCollection<{dtoFullName}>("{typeKey}")[rootIndex];
    return new NormalizedResult<{dtoFullName}>(root, rootIndex, context);
}
```

After:
```csharp
public static {containerFullName} Normalize({rootType} source)
{
    var context = new NormalizationContext({typeCount});
    var rootIndex = Normalize{TypeName}(source, context);
    var result = new {containerFullName}();
    result.RootIndex = rootIndex;
    // Entity list population for each type in allNodes:
    var __{camel}Col = context.GetCollection<{DtoFullName}>("{typeKey}");
    var __{camel}Arr = new {DtoFullName}[__{camel}Col.Count];
    for (var __i = 0; __i < __{camel}Col.Count; __i++)
        __{camel}Arr[__i] = __{camel}Col[__i];
    result.{TypeName}List = __{camel}Arr;
    // ... repeat for each type ...
    return result;
}
```

Where `containerFullName` = namespace + `Normalized{rootNode.TypeName}Result`.

**Critical details:**
- Entity list population uses `GetTypeKey()` for the context lookup key (handles `WithName()` custom keys), but `{TypeName}List` for the container property name (always uses TypeName, not custom key)
- Must iterate ALL nodes (including root type) for entity list population
- Do NOT skip or modify `NormalizeXxx()` helper methods — all stay as-is
- Do NOT modify `EmitNormalizeHelper()` — it's unchanged

**Test updates:**
- Change assertions from `NormalizedResult<TestApp.NormalizedPerson>` to `TestApp.NormalizedPersonResult`
- Check for `result.RootIndex = rootIndex;` instead of `new NormalizedResult<...>(...)`
- Check for entity list population lines
- Add test: multi-root emits separate container types
- Add test: WithName() uses custom key for context lookup but TypeName for property name

### Task 4: Update NormalizeGenerator pipeline

**Files:**
- Modify: `src/DataNormalizer.Generators/NormalizeGenerator.cs` (lines 95-143)

**Changes to TransformConfig():**

After emitting per-type DTOs (line 113-129 — unchanged, ALL types get DTOs including root), add container emission:

```csharp
// Emit Container classes (one per root type)
foreach (var rootType in model.RootTypes)
{
    ct.ThrowIfCancellationRequested();
    TypeGraphNode? rootNode = null;
    for (var i = 0; i < allNodes.Count; i++)
    {
        if (allNodes[i].TypeFullName == rootType.FullyQualifiedName)
        {
            rootNode = allNodes[i];
            break;
        }
    }
    if (rootNode == null)
        continue;
    var containerSource = ContainerEmitter.Emit(rootNode, allNodes, model.JsonNamingPolicy);
    var containerHintPrefix = string.IsNullOrEmpty(model.ConfigNamespace)
        ? model.ConfigClassName
        : $"{model.ConfigNamespace}.{model.ConfigClassName}";
    var containerNs = EmitterHelpers.GetNamespace(rootNode.TypeFullName);
    var containerHintName = string.IsNullOrEmpty(containerNs)
        ? $"{containerHintPrefix}.Normalized{rootNode.TypeName}Result.g.cs"
        : $"{containerHintPrefix}.{containerNs}.Normalized{rootNode.TypeName}Result.g.cs";
    sources.Add(new GeneratorSourceEntry(containerHintName, containerSource));
}
```

**Important:** Do NOT use LINQ (`FirstOrDefault`, `Where`) — `NormalizeGenerator.cs` doesn't import `System.Linq`. Use explicit for-loops (matching existing patterns in the file).
