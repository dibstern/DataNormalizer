# Tasks 4 & 5: NormalizeGenerator Pipeline + Delete NormalizedResult -- Audit Report

**Summary:** Task 4 has several code-level issues: an undefined variable in the code snippet, a missing consideration for self-referential root types that need NormalizeXxx helpers, and incomplete wiring instructions for passing root type info to emitters. Task 5 has a contradiction between the design doc (remove `CastingReadOnlyList<T>`) and the implementation plan (keep it), plus missing coverage of `NormalizationContextTests` and documentation files referencing `NormalizedResult`.

## Critical Issues

**Finding #2 is the highest-risk issue.** The plan says to skip `NormalizeXxx()` helpers for root types, but self-referential root types (TreeNode with `List<TreeNode> Children`) NEED that helper for recursive normalization. The existing circular reference E2E test uses exactly this pattern and would break. This needs a design decision before implementation.

**Finding #7 is related** -- if the root type's per-type DTO is no longer emitted (because it's replaced by the container), but the `NormalizeXxx()` helper still runs and calls `GetOrAddIndexAndStore<DTO>()`, what DTO type is used for dedup? The container (no `IEquatable`) or the original per-type DTO (which no longer exists)?

The likely correct resolution: **always emit per-type DTOs for ALL types including root**, and ADDITIONALLY emit the container. The container aggregates the results but doesn't replace the per-type DTO for internal normalization purposes. The root's per-type DTO would still be used during normalization for dedup, just not exposed as a separate output file... Actually wait, it IS an output file for the DTO class definition. The plan should be amended to: only skip the root DTO emission when the root type is NOT referenced by any other type in the graph (i.e., pure wrapper types).
