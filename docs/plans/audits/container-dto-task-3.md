# Task 3: Modify DenormalizerEmitter to accept container — Audit Report

**Summary:** Task 3 has several significant gaps. The plan's description of changes is high-level and misses critical details around how the root type is separated from non-root types in `allNodes`, the `.Count` to `.Length` transition in Pass 1/Pass 2, the `WithName()` / custom name interaction with container property names, and the complete lack of root reconstruction code specification for all property kinds and collection types.

## Summary by Category

**Incorrect Code:** The plan underspecifies root reconstruction (Finding #3) and misses the `WithName()` interaction with container property naming (Finding #4). Both need concrete code patterns added to the plan.

**Missing Wiring:** Root type exclusion from entity list iteration is unaddressed (Finding #2, #5). Container type name computation is implicit but happens to work (Finding #7).

**Implicit Assumptions:** The claim that "Pass 1 and Pass 2 are unchanged" is FALSE — `.Count` must become `.Length` (Findings #1, #6). Self-referencing root types create an ambiguity (Finding #12).

**Insufficient Test Coverage:** Three entirely new code paths (root reconstruction for nullable refs, collections, and simple properties) have no specified tests (Findings #8, #9, #10).

**No issues found in:** State Issues, Non-Strict Typing.

---

## Critical Path Summary

The **highest-risk items** that will cause implementation to fail or produce incorrect generated code:

1. **`.Count` → `.Length` (Finding #1)** — Generated code will not compile since arrays don't have `.Count`.
2. **Root type in `allNodes` (Finding #2)** — `EmitGetCollections` will try to access `normalized.{RootTypeName}List` which doesn't exist on the container.
3. **Root reconstruction completeness (Finding #3)** — Without handling all collection kinds, the generated denormalizer will be incomplete for root types with non-array collection properties.
4. **Self-referencing root edge case (Finding #12)** — Needs a design decision before implementation.
