# DocFX Documentation Site Design

**Date:** 2026-03-20
**Status:** Approved

## Goal

Add auto-generated API reference documentation and conceptual articles for the DataNormalizer NuGet library, hosted for free on GitHub Pages via DocFX.

## Audience

Library consumers — developers who install the NuGet package and need API reference plus usage guides.

## Approach

DocFX + GitHub Pages. DocFX is the official .NET Foundation documentation tool. It generates static HTML from XML doc comments and markdown articles. GitHub Pages hosts the output for free.

## File Structure

```
docs/
├── docfx.json              # DocFX configuration
├── toc.yml                 # Top-level table of contents
├── index.md                # Landing page (adapted from README)
├── articles/
│   ├── toc.yml             # Articles navigation
│   ├── getting-started.md  # Quick start guide
│   ├── configuration.md    # Configuration options deep-dive
│   └── diagnostics.md      # DN0001-DN0004 explanations + resolution
├── api/
│   ├── toc.yml             # Auto-generated by DocFX
│   └── index.md            # API reference landing page
└── _site/                  # Generated output (gitignored)
```

The existing `docs/plans/` directory is internal and will not be published.

## DocFX Configuration

- **API metadata source:** `src/DataNormalizer/DataNormalizer.csproj` (runtime library only). The source generator is all `internal` — nothing to document publicly.
- **Theme:** Default DocFX modern theme.
- **Output:** `docs/_site/` (added to `.gitignore`).

## Conceptual Articles (Initial Set)

1. **Getting Started** — Installation, quick example, basic usage.
2. **Configuration Guide** — AutoDiscover, Inline, IgnoreProperty, ExplicitOnly mode, multiple root types.
3. **Diagnostics Reference** — DN0001–DN0004 with severity, description, and resolution.

Content is extracted and refined from the existing README.

## Landing Page

Adapted from the README. Includes project description, before/after example, and links to articles and API reference.

## GitHub Actions Workflow

New file: `.github/workflows/docs.yml`

- **Trigger:** Tag push (`v*`) — docs deploy alongside NuGet releases.
- **Steps:**
  1. Checkout code
  2. Install .NET SDK 8.x
  3. Install DocFX (`dotnet tool update -g docfx`)
  4. Run `docfx docs/docfx.json`
  5. Upload `docs/_site` via `actions/upload-pages-artifact@v3`
  6. Deploy via `actions/deploy-pages@v4`
- **Permissions:** `pages: write`, `id-token: write`
- **Concurrency:** Single deployment, no cancellation of in-progress runs.

## GitHub Pages Setup

- Enable GitHub Pages in repo settings: Source = "GitHub Actions"
- Site URL: `https://dibstern.github.io/DataNormalizer/`
- Update `PackageProjectUrl` in `.csproj` to match actual GitHub URL

## Decisions

| Decision | Rationale |
|----------|-----------|
| DocFX over MkDocs/Jekyll | First-class .NET support, understands XML doc comments natively |
| Deploy on tags only | Docs match released versions, not every commit |
| Only document runtime library | Source generator is all internal types |
| Default theme | Professional, familiar to .NET devs, zero maintenance |
| docs/ subfolder (not repo root) | Keeps docfx config separate from project root |

## Out of Scope (for now)

- Versioned docs (multiple versions side-by-side)
- PDF generation
- Custom theme
- CHANGELOG / release notes on the docs site
- Contributor / architecture documentation
